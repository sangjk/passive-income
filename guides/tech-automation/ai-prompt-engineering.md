## Step 1: Master Prompt Engineering

### Core Prompting Techniques

| Technique | When to Use | Example |
|-----------|------------|---------|
| **Zero-shot** | Simple tasks | "Summarize this article" |
| **Few-shot** | Need examples | "Here are 3 examples of [X]. Now do [Y]." |
| **Chain-of-thought** | Complex reasoning | "Think step by step..." |
| **Role-playing** | Specific expertise | "You are a senior [role]..." |
| **System prompts** | Consistent behavior | "Always respond in [format]..." |
| **Template-based** | Repeatable outputs | "Fill in this template: [structure]" |
| **Self-critique** | Quality improvement | "Review your answer and improve it" |
| **Decomposition** | Complex tasks | "Break this into steps, then solve each" |

### Prompt Engineering Framework

```
1. CONTEXT — What's the situation? Who is the audience?
2. ROLE — What expert persona should the AI adopt?
3. TASK — What specifically should the AI do?
4. FORMAT — How should the output be structured?
5. CONSTRAINTS — What should the AI avoid or limit?
6. EXAMPLES — Show the AI what good output looks like
```

### Advanced Techniques

1. **Prompt chaining** — Output of one prompt feeds into the next
2. **Meta-prompting** — Use AI to write better prompts
3. **Ensemble prompting** — Multiple prompts, combine results
4. **Constitutional AI** — Built-in safety and quality checks
5. **Retrieval-augmented** — Combine prompts with external data

## ⚠️ Common Mistakes

1. **Not testing prompts** — Always test across different inputs
2. **Being too generic** — Specific prompts outperform vague ones
3. **Ignoring model differences** — GPT-4 ≠ Claude ≠ Gemini
4. **Not documenting** — Clients need to understand and modify prompts
5. **Underpricing** — Good prompts save businesses thousands; price accordingly
